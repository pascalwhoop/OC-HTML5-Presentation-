Vorstellung der Marktteilnehmer

Amazon AWS IoT

  • HTTP & MQTT
  • Bi-direktional
  • Client Libraries + Gateways
  • "Besonders": Device Shadows & IoT API

Besonders in Anführungsstrichen, da Amazon diese features besonders betont, sie jedoch nicht allzu ungewöhnlich sind.

Device Shadows ist ein Marketing Begriff für Queueing und Caching

Azure IoT Hub

admin zugriff 30 min max 3 leute

für MS ist safe harbor irrelevant, da andere Zertifizierung

  • HTTP, MQTT & AMQP
  • Bi-direktional
    • Hot path + cold path D2C
    • TTL C2D Commands
  • Client Libraries + Gateways
    • C for many Linux distributions, Windows, and RTOS
    • managed languages such as C#, Java, and JavaScript

IBM Bluemix

  • MQTT only
    • firewall issues
  • Bi-direktional
  • Client Libraries, no Gateways
    • Client libraries auf Github Open-Sourced

Oracle Cloud

  • HTTP(S) only
  • Bi-Direktional
    • device-side polling
  • Client libraries
    • aktuell nur Java (1.5)

Technisches Intermezzo

Fazit

  • Es gibt keinen idealen IoT Cloudanbieter

  • Alle Anbieter setzen auf REST und bieten APIs an

  • Lizenzmodelle stark verschieden, kann abhängig vom Projekt starke Einflüsse auf Kosten haben

  • Oracle muss insbesondere bzgl. Endgerätesupport nachholen. Es gibt eine Welt abseits von Java

  • Open-Source / Community Ansatz bei heterogener Gerätemarktstruktur unablässig
